# Handoff: 온더볼 (On the Ball) — 해외축구 가십 & 밸런스 모바일 앱

## Overview

**온더볼**은 해외축구의 가십·밸런스 대결·퀴즈를 다루는 **모바일 전용** 커뮤니티 투표 앱입니다.
핵심 콘텐츠 타입 4가지:

1. **밸런스 게임** (양자택일) — "메시 vs 호날두", 유니폼 A vs B 등. 투표 → 실시간 비율 공개
2. **랭킹 투표** — 발롱도르, EPL 올해의 선수 등 다지선다 1인 투표
3. **퀴즈** — 라인업 국적만 보고 팀/시즌 맞히기, TMI 4지선다 등. 매일 1문제, 연속 정답 streak
4. **TMI 진실/거짓** — 선수 가십 카드를 스와이프로 판정, 다수 의견과 비교

이 번들은 그 1차 프로토타입(v1)의 디자인 레퍼런스입니다.

## About the Design Files

`prototype/` 폴더의 파일들은 **HTML로 만든 디자인 레퍼런스**입니다 — 의도된 룩과 인터랙션을 보여주는 프로토타입이며, 그대로 복사해 쓸 프로덕션 코드가 아닙니다.

**할 일: 이 HTML 디자인을 대상 코드베이스의 기존 환경(React Native, Flutter, Next.js 등)에서, 그 코드베이스의 기존 패턴과 라이브러리로 재구현하는 것.** 환경이 아직 없다면 이 프로젝트에 가장 적합한 프레임워크를 선택해 구현하세요. (모바일 전용 웹이라면 Next.js + 모바일 뷰포트 고정, 네이티브 앱이라면 React Native/Expo 권장.)

프로토타입 확인 방법: `prototype/온더볼.html`을 브라우저에서 열면 iPhone 프레임 안에서 전 화면을 클릭해볼 수 있습니다. (CDN으로 React 18 + Babel Standalone을 로드하므로 인터넷 연결 필요.)

주의: 프로토타입의 `ios-frame.jsx`(iPhone 베젤)와 바깥 스테이지 배경은 **프레젠테이션용 장치**입니다. 실제 구현 대상이 아닙니다. 구현 대상은 프레임 *안쪽*의 앱 UI입니다.

## Fidelity

**High-fidelity (hifi).** 컬러·타이포·간격·인터랙션이 최종 의도값입니다. 아래 명세와 프로토타입 소스의 수치를 그대로 재현하되, 구현은 대상 코드베이스의 컴포넌트 시스템으로 하세요.

단, 다음은 **플레이스홀더**입니다:
- 선수 실루엣 SVG (`PlayerSilhouette`) → 실제 선수 컷아웃 사진으로 교체 예정
- 유니폼 SVG (`ShirtSVG`), 클럽 크레스트 → 실제 이미지로 교체 예정
- 모든 데이터(투표수, 댓글, 통계)는 목데이터 → 실제 API 연동 필요

## Design System: "Tifo" (Supabase Feel)

바인딩 디자인 시스템. 전체 토큰은 `prototype/design/colors_and_type.css`에 CSS 변수로 정의되어 있습니다. 핵심 규칙:

- **한 뷰포트당 컬러 이벤트는 에메랄드(`#3ecf8e`) 1개.** 나머지는 잉크 그레이 래더
- 에메랄드 위 텍스트는 **항상 `#171717`** (흰색 금지)
- 디스플레이 폰트 웨이트는 **500 고정** (600 이상 금지) + 네거티브 트래킹
- 버튼은 **6px 라운드** (pill 버튼 금지) — 프로토타입의 `--radius-sm` 참조
- 그림자 대신 **1px 헤어라인**이 카드 구조를 담당. resting 상태는 flat
- 배경 그라데이션·블러·글래스모피즘 금지 (하단 탭바만 예외적으로 blur 사용)
- 애니메이션 이징: `cubic-bezier(0.2, 0, 0, 1)`, 150–350ms. 바운스/스프링 금지
- 강한 컬러(클럽 컬러, 국기)는 **콘텐츠**로만 허용 — 크롬(버튼, 네비)에는 금지

## Screens / Views

앱 셸: 상단 앱바 + 스크롤 영역 + **하단 플로팅 탭바 5개** (홈 / 밸런스 / 퀴즈 / TMI / 내 활동).
탭바: `rgba(23,23,23,0.92)` + blur, radius 28px, 좌우 12px 플로팅, 활성 탭은 흰 텍스트 + 에메랄드 아이콘.
디테일 화면(밸런스 상세, 퀴즈 상세, 랭킹, 유니폼 투표)은 탭 위에 풀스크린 모달로 덮입니다. 아이콘: **Lucide** 1.5px 스트로크.

### 1. 홈 (`screen-home.jsx`)

위→아래 순서:
1. **앱바** — 좌측 워드마크(에메랄드 볼 + "온더볼"), 우측 streak 칩(`12일 연속`, 다크 pill) + 검색/알림 아이콘 버튼
2. **오늘의 밸런스 히어로** — 태그(uppercase, mono 10px) → 26px/600 헤드라인 → 메타(진행 중 pill + 투표수 + 마감) → **대각선 스플릿 카드** (아래 상세) → "한 표 던지기" 다크 버튼 + "결과" 보조 버튼
3. **가벼운 양자택일 캐러셀** — 가로 스크롤, 240px 카드. 상단 절반이 다크/소프트 2분할 + 중앙 VS 배지
4. **오늘의 퀴즈 다크 배너** — `#1c1c1c` 카드, 에메랄드 mono 라벨 "▸ 오늘의 퀴즈", 국기 4줄 프리뷰, "맞춰보기 →" 에메랄드 버튼
5. **진행 중인 투표** — 80px 커버 썸네일(트로피/유니폼) + 제목 + 투표수 리스트 카드
6. **지금 뜨거운 떡밥** — 랭킹 리스트 (mono 순번, 제목, 투표수, 델타 pill: 상승 에메랄드/하락 크림슨/NEW)
7. **TMI 프로모 카드** — `#fffdf5` 배경, 🔥 아이콘 (콘텐츠 컬러로서의 옐로)

### 대각선 스플릿 카드 (밸런스 게임의 시그니처)

- 컨테이너: aspect-ratio 4/5 (홈) 또는 3/4 (디테일), radius 18px, overflow hidden, 세로 flex
- 위 절반 = 선택지 A: `clip-path: polygon(0 0, 100% 0, 100% 100%, 0 58%)`
- 아래 절반 = 선택지 B: `clip-path: polygon(0 42%, 100% 0, 100% 100%, 0 100%)`
- → 두 절반의 경계가 **지그재그 대각선**을 형성 (첨부 레퍼런스 "버거 보트"의 에너지를 모노톤으로 번역)
- A면은 다크(`#0a0a0a`~`#1c1c1c`) + 흰 텍스트, B면은 화이트 + 잉크 텍스트 (또는 콘텐츠 컬러 — 클럽 컬러 허용)
- 각 면: A/B 뱃지(12px), 라틴 서브라벨(mono 10px uppercase op.55), **거대 이름 타입** (46–56px/700/-2px tracking), 메타 12px
- 반대 코너에 선수 실루엣 플레이스홀더 (opacity 0.08–0.13)
- 중앙: **VS 배지** — 64px 원, 에메랄드, `rotate(-12deg)`, 4px 흰 보더, 등장 시 pop 애니메이션(scale 0.8→1.1→1)
- 탭 시: 선택 면 scale(1.01) + 반대 면 opacity 0.42, 380ms 후 결과 화면 전환

### 2. 밸런스 리스트 (`screen-balance.jsx` — `BalanceListScreen`)

- 28px/700 타이틀 "밸런스 게임" + 서브카피
- 필터 pill 가로 스크롤 (전체/GOAT/이적/유니폼/스쿼드 대결/리그) — 활성은 다크 pill
- 리스트 카드: 상단 70px 2분할 스트립(A 다크 좌 / B 우) + 중앙 30px VS 배지 + 태그/마감 pill + 제목 + **4px 비율 바** (다크 = A 점유율) + 양측 투표수

### 3. 밸런스 디테일 (`BalanceDetail` — 모달)

**투표 전** (`BalanceVoteView`):
- SubHeader (뒤로가기 + 카테고리 타이틀 + 공유)
- 메타 pill 줄 → 24px 헤드라인 → 서브카피
- 3/4 대각선 스플릿 (탭 = 투표)
- 안내문 "한쪽 카드를 탭하면 즉시 한 표 반영돼요."
- 인포 스트립: "결과 그래프는 투표한 사람만 볼 수 있어요. 댓글은 결과 공개 후에 열려요." — 이게 핵심 UX 규칙

**투표 후** (`BalanceRevealView`, 380ms 후 전환):
- "결과 공개" pill + 내 선택 칩
- **비율 바** — 56px 높이, A/B 톤으로 채운 두 블록, 퍼센트 22px/700, width 0.8s ease 애니메이션, WIN pill, 내 표 마커
- 스탯 비교 그리드 (3행: 통산 골/발롱도르/우승컵 등, 중앙 헤어라인)
- 양측 blurb 카드 (승자는 2px 에메랄드 보더)
- **응답자 분석** — 연령대별 8px 가로 스택바 3줄
- **한 줄 거들기** — 댓글 입력 + 댓글 리스트 (아바타, 팬 태그 pill, 동감 카운트)

### 4. 퀴즈 리스트 (`screen-quiz.jsx` — `QuizListScreen`)

- 타이틀 + "매일 오전 8시, 새 문제가 한 개씩 열려요."
- **Streak 카드** — 다크, 56px 에메랄드 원 안에 streak 일수, "상위 4% · 한 번이라도 틀리면 0으로 리셋"
- 오늘의 문제 (NEW pill + 정답률/도전자 수)
- 예정된 문제 (dashed 보더 + 잠금 아이콘)
- 지난 문제 보관함 (정답/오답 아이콘 리스트)

### 5. 퀴즈 디테일 (`QuizDetail` — 모달)

- 메타(정답률 %, 도전자 수) → 22px 질문 헤드라인
- **피치 뷰** — 3/4 그린 그라데이션(`#2d7a4f→#245d3d`) 피치, 흰 라인 프레임 + 센터서클, 잔디 스트라이프. 포메이션 4줄(GK 하단→FW 상단, `column-reverse`)에 **국기+포지션 칩** (다크 pill, 국기 20×14 + mono 포지션 라벨)
- 캡션: "4-3-3 · 11 caps · 7 nations" (mono)
- **힌트 토글** — dashed pill 버튼 → 옐로 카드 (`#fff8c5`)
- **4지선다** — 팀명 + 시즌(mono). 정답 후: 정답 에메랄드 틴트+보더 / 내 오답 크림슨 / 나머지 dim + 각 선택지 선택률 표시
- **리빌 카드** — 정답 시 다크 카드 "정답이에요 ⚽" / 오답 시 화이트 "아쉽다." + 정답 명기 + 다시 도전/공유 버튼
- **스트릭 업데이트 카드** — 정답: 13으로 증가 / 오답: "연속 정답이 0으로 리셋"

### 6. 랭킹 투표 (`screen-ranking.jsx` — `RankingDetail`, 모달)

- **다크 히어로** — 옐로 mono 라벨 "BALLON D'OR · 2026" + 타이틀 + 총 투표수 + 옐로 트로피 아이콘
- 후보 리스트 (radio 선택): 순번(mono) + 아바타(파스텔 원 + 국기 뱃지) + 이름 + 클럽
- **하단 고정 CTA 스트립** — 탭바 위에 떠 있음. 미선택 시 "한 명을 골라주세요" disabled, 선택 시 "{이름}에 한 표"
- 투표 후: 득표순 정렬 + 5px 비율 바 (1위 에메랄드, 내 선택 잉크, 나머지 그레이) + 퍼센트/득표수 + "지역별 1위" 2×2 그리드

### 7. 유니폼 투표 (`KitVoteDetail` — 모달)

- 2열 그리드 유니폼 카드 — 1:1 소프트 배경에 유니폼 SVG(클럽 컬러, sash/호리즌탈 스트라이프 변형)
- 탭 = 투표 (재탭 = 취소). 투표한 카드는 에메랄드 보더
- 투표 후 전체 카드에 비율 바 + 퍼센트 공개

### 8. TMI (`screen-tmi.jsx` — 탭)

- 프로그레스 바 (완료 다크 / 현재 에메랄드 / 남음 헤어라인)
- **카드 스택** — 현재 카드 + 뒤 카드 peek(translateY(12px) scale(0.96) op 0.6)
- 카드: 선수 아바타 + 국기 + 클럽 → "TMI · 떡밥" mono 라벨 → 22px 주장 텍스트 → "출처 미상 —" 디테일 박스
- 하단 판정 버튼 2개: **거짓**(화이트+크림슨 보더) / **진실**(에메랄드)
- 판정 시: 카드가 좌(거짓)/우(진실)로 날아가는 600ms 트랜지션 → 다음 카드
- 판정 후 카드에 진실/거짓 비율 바 (에메랄드 vs 다크) + 내 선택이 다수/소수 의견인지 표시
- 덱 완료: 다수 의견 일치 수 요약 + 다시 풀기

### 9. 내 활동 (`screen-activity.jsx` — 탭)

- 프로필 (클럽 그라데이션 아바타 + 팬 태그 + 가입일)
- 4열 스탯 (투표/퀴즈/연속/정답률)
- 뱃지 3열 그리드 (미획득은 op 0.55 + `?`)
- 최근 한 표 리스트 (콘텐츠 타입 아이콘 + 내 선택 + 다수 의견 일치율)
- **나의 축구 성향** 다크 카드 — "낭만파 — ..." 인용문, 60일 데이터 기반, 월 1회 갱신

## Interactions & Behavior

- 화면 전환: 탭 전환은 즉시, 디테일은 풀스크린 모달 오버레이
- 밸런스 투표: 탭 → 380ms(선택 강조) → 결과 뷰. 비율 바 width 0.8s `cubic-bezier(0.2,0,0,1)`
- 투표 정책 카피: "한 번 투표하면 24시간 안에 한 번만 바꿀 수 있어요."
- 결과 게이팅: 결과/댓글은 투표 후에만 노출
- TMI 스와이프: 버튼 판정(스와이프 제스처는 실구현에서 추가 권장), 카드 이탈 `translateX(±120%) rotate(±14deg)` 600ms
- 등장 애니메이션: `fade-up` 350ms (opacity 0→1 + translateY 8px→0)
- `prefers-reduced-motion` 존중 (트랜지션 0ms)
- 스크롤 영역 하단 패딩 122px (플로팅 탭바 + 홈 인디케이터 확보)

## State Management (구현 시 필요한 상태)

- `tab`: 'home' | 'balance' | 'quiz' | 'tmi' | 'me'
- `detail`: null | { type: 'balance'|'quiz'|'ranking'|'kit', id }
- 밸런스: `vote`(null|'a'|'b') + `reveal`(bool)
- 퀴즈: `state`('playing'|'done') + `picked` + `hintOpen`
- 랭킹: `picked` + `voted` (투표 후 득표순 재정렬)
- TMI: `idx` + `votes`({idx: 'true'|'false'}) + 카드 이탈 애니메이션 상태
- 실서비스: 유저 투표 이력, streak, 뱃지는 서버 상태

- **Pretendard Variable** woff2 — `prototype/design/fonts/` 동봉 (SIL OFL)

- 국기 — 프로토타입에선 인라인 SVG 17개국. 실구현에서는 flag 라이브러리/스프라이트 권장
